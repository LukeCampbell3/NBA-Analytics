What This System Is Doing

This engine is a mechanism-first NBA analysis system. Instead of ranking players only by single-number leaderboards, it tries to answer: what value the player creates, why it exists, and how well it transfers to another team environment.

The core idea is to place each player in a context bubble: we separate what appears to be intrinsic skill from what is caused by deployment, teammates, schedule strength, and scheme. Then we run constrained counterfactuals (e.g., “Best team fit”) that are forced to remain realistic.

You should think of this as: explain → isolate → translate → constrain → sanity-check → grade confidence.

The “Isolation” Process (Step-by-Step)

1) Start from a Player Card

Each player card is a structured summary of: box production, advanced rates, shot profile, creation profile, role identity, comparables, and trust/uncertainty signals. The card is the consistent input interface so every tool uses the same evidence base.

2) Decompose Observed Production into Intrinsic + Context

We treat observed outcomes as a mixture of what the player is and what the environment does to the player:

P_observed = P_intrinsic + P_context + ε
  • P_intrinsic: signals that should travel across teams (skill + stable tendencies).
  • P_context: teammate help/hurt, scheme usage, schedule strength, lineup roles.
  • ε: noise (variance, missing variables, partial observability).

The system does not claim perfect isolation; it is an accounting model that assigns directionality (suppressed vs boosted) and exposes uncertainty.

3) Build a Team Environment Model (from your dataset)

The engine builds team “environment profiles” from the full card set: spacing/assist ecosystem, creation share, finishing friendliness, and context adjustment proxies. This is how we evaluate whether a player’s mechanisms are likely to be utilized or wasted on a given team.

4) Run a Counterfactual Projection (but with clamps)

We estimate a best-case role translation for each candidate team and then enforce feasibility constraints (clamps) so projections don’t become fantasy.

Maximize ΔValue(team, role) subject to role ∈ C (clamps: usage/minutes/shot mix/defense/contract feasibility)
  • Outputs include the best achievable role and exact clamps that fired.
  • If a role expansion is unrealistic, the scenario is automatically reduced and flagged.

5) Add Reality Anchors (anti-drift)

Even internally consistent models can drift from real impact. Two anchors reduce false confidence:

  • Impact priors (EPM / LEBRON / DARKO / RAPM family): compare your conclusions to an independent impact frame.
  • Defense portability: estimate what a player can defend and where they break when targeted (playoff survivability).

If the anchors disagree with a strong claim, the system flags it for review and reduces confidence.

6) Grade the Claim with a Confidence Trace

Every major conclusion is graded, not asserted. Confidence is computed from multiple components so it can be inspected.

Confidence = f(data_quality, sample_support, extrapolation, clamp_pressure, comp_density, anchor_agreement, isolation)

The Pillars (Identity of the System)

Pillar 1

Value is a decomposition, not a number

Players are represented as a mechanism vector (on-ball, off-ball, spacing, creation, finishing, POA defense, rim, etc.), then weighted for team needs and playoff context.

Pillar 2

Intrinsic vs deployment separation

The system explicitly separates what appears to be player skill from what the team environment contributes (help/hurt, scheme emphasis, schedule strength).

Pillar 3

Counterfactuals are constrained by clamps

Every “Team X fit” projection is forced through feasibility constraints and reports which constraints fired. This prevents unrealistic role inflation.

Pillar 4

Confidence governance is part of the product

Confidence is a traceable result tied to evidence quality, isolation, and anchor agreement—so outcomes are graded, not overconfident.

Pillar 5

Comparables are conditional (comp density)

The system measures how comparable a player is. Low comp density means a unique role and wider uncertainty; the system penalizes forced comps and forced conclusions.

Pillar 6

Reality anchors: impact priors + defense portability

The engine self-audits using independent impact priors and playoff-aware defensive portability clamps. Disagreement triggers review flags and confidence reduction.

Pillar 7

Valuation: contracts + aging curves

Value becomes decision-grade when projected contribution is compared to contract cost and adjusted by archetype aging curves. This supports surplus value and time horizon reasoning.

Pillar 8

Outputs encode the identity

Every player output must show identity, decomposition, counterfactual + clamps, anchors, risk, and confidence trace. If outputs don’t show the work, the system’s identity blurs.

Core Scores On Player Cards (What They Mean)

Player Value

A normalized single-number value score intended for cross-player comparability. It is a rollup of the value vector, anchored by impact priors (when available) and adjusted for role/deployment.

Breakout Score

A context-aware upside score that requires both a believable growth path and feasible translation. It is restricted by clamps, comp density uncertainty, and anchor disagreement.

Breakout Likelihood

A confidence-aware category (Low / Medium / High) derived from the calibrated breakout probability and evidence quality. High likelihood cannot occur if the scenario is heavily clamped or poorly supported.

Team Fit Context (%)

A fit score estimating how well a player’s mechanism profile aligns with a team environment profile. It measures whether the team will utilize or suppress the player’s strengths.

  • >= 8%: strong / excellent fit edge
  • 6% to 8%: good fit path
  • < 6%: weak fit path

Mechanism Metrics (Why A Scenario Works)

Suppression Relief

A measure of how much a new team/system is expected to unlock value that is currently underutilized. High suppression relief means: the player has a strong skill signal, but the current team environment does not emphasize it.

Spacing Gravity Skill

Off-ball shot gravity pressure estimated from 3PT volume, 3PT efficiency, and assisted/catch-and-shoot profile. This captures how much defensive attention the player pulls away from primary actions (even if box stats stay modest).

Defense Portability

A survivability estimate: will defensive value translate across matchups and survive targeting? It captures role identity (POA/wing/big), screen navigation stress, switchability, size mismatch risk, and foul risk. Portability becomes a clamp: if portability is low, certain counterfactual roles are not feasible in playoff contexts.

Clamp Pressure

How restricted the projection is by feasibility constraints (usage/minutes/shot mix/defense role). High clamp pressure means the “upside path” is blocked by role reality.

Prior Disagreement (Anchor Drift)

A disagreement signal between your model’s conclusion and independent impact priors (or the spread across multiple priors). Higher disagreement increases uncertainty and triggers review flags for strong claims.

Comp Density

Similarity density from comparable players. Low comp density indicates a unique profile and wider uncertainty bounds. The system penalizes strong conclusions when comp density is low.

Team Contribution Channels (How Value Shows Up)

Scenarios decompose contribution rather than relying on one lever:

  • Role Stability: minutes feasibility + continuity (can the role exist without distortion?)
  • Creation Lift: usage headroom + on-ball creation translation
  • Spacing / Off-ball Lift: gravity + scheme utilization (value without box inflation)
  • Defense / Portability: matchup survivability, switch targets, foul risk, and playoff clamps
  • Anchor Alignment: impact priors agree/disagree with the claim (anti-drift)

Composite contribution strength is used as a gate: weak channels cannot produce high-confidence conclusions.

How Breakout Is Calculated (High Level)

Visible Breakout (box/role)

VisibleBreakout ~= f(usage_headroom, minutes_headroom, fit_edge, translation_quality) - penalties(clamp_pressure, extrapolation, isolation, comp_density, anchor_disagreement)

Ecosystem Breakout (hidden/non-box)

EcosystemBreakout ~= f(hidden_skill_total, system_utilization_uplift, suppression_relief) - penalties(role_instability, isolation, low evidence, anchor_disagreement)

Final breakout decision

p_any_breakout = Calibrate( max(p_visible_breakout, p_ecosystem_breakout) )

“Calibrate” means confidence is trained to match empirical hit rates in backtests, so it remains honest.

Shot Model Views (What You’re Seeing)

Volume

Zone-level shot share (rim / mid / 3) distributed across regions using the player’s shot profile priors.

FG%

Zone-level shooting efficiency constrained by broad profile + location priors.

Shot Value

Expected points per shot by region, combining FG% with 2PT/3PT value structure.

True Impact (Shot Contribution View)

Shot value adjusted by contextual possession factors. Lower-confidence zones are visually faded so users do not over-trust them.

Confidence and Uncertainty (Why It Matters)

Confidence is not upside. It is the reliability of the claim given: evidence quality, isolation, extrapolation, clamp pressure, and anchor agreement.

The system is designed to say:

  • Validated: plausible + supported + calibrated confidence
  • Plausible but not validated: coherent story but missing support or constrained by clamps
  • Flag for review: internally consistent but disagrees with anchors (impact priors / defense portability)

In visuals, uncertainty is shown through alpha/intensity changes so modeled zones are not mistaken for equally observed outcomes.

The “Airtight” Plan (How This Becomes Defensible)

  1. Glossary contract: define each metric (inputs, scale, interpretation, failure modes) so the system stays legible.
  2. Standardize outputs: one JSON schema for every tool (header, signals, confidence trace, warnings, evidence).
  3. Backtest calibration: define breakout labels operationally and calibrate confidence to empirical hit rates.
  4. Measure drift: track disagreements vs anchors and detect systematic bias by archetype/team/role.
